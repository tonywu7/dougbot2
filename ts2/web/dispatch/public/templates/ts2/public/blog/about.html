What is this?

Yet another Discord bot for the DougDoug Discord server.

We've been using @Frank for quite a while now, and its custom commands are awesome (YAGPDB is my main source of inspiration as you can tell from the website layout).

But its drawback is also in the name — being a general-purpose bot means it doesn't integrate deeply with the server nor offer truly unique customization, nor can it expend a lot of computing power for us.

With @Frank, there's no way to, for example:

  • make external web requests;
  • find messages in bulk;
  • store arbitrary information to be used later;
  • integrate with Twitch in ways more than merely notifications;

... and I want something more powerful.
(I also got used to Jinja and writing custom commands in Go is just a pain in the ass.)

In addition to a much better help command, more expressive and friendly error messages, and a more configurable command permission system (all in my opinion, of course), there is now support for custom timezones and a command to join multiple messages together (a limited number of people will know what this could be for.)

Writing an in-house bot from scratch also paves the way for two special features down the road: {% featurelink 'district-soundfinder' %} and {% featurelink 'district-encyclopedia' %} (and as many other features as I can humanly-possibly implement). Feel free to make suggestions through the feedback form!

What tools do you use?

The bot is written in Python using discord.py, but organized as Django apps (which also drives this website). It is hosted on AWS.

Interactive stuff on the webpage are written using Vue in TypeScript and styles are written in Sass.

Can I run this bot?

Yes.

The bot as it is currently hosted on {{ CURRENT_DOMAIN }} is only usable in the DougDoug Discord server — you'll notice that you can't join the bot anywhere else even if you had logged in. But nothing stops you from forking the bot's source code, making a Discord app, and hosting the bot yourself.

In fact, one of the initial design choices I had for the bot is that it should be as versatile and easy to use as possible, which is also the reason there is the control panel on the website, and login is done using Discord accounts. The bot does not need a frontend to function, but having a frontend makes it easy for people (your peer moderators, for example) to setup the bot without dealing with all the underlying moving parts.

With that being said, you do need to know how to use Docker. Follow the README on GitHub and file issues for any bug you find!
